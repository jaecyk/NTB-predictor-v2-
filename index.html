Step 1 — Add these 3 files to your GitHub repo
Download the zip below. It contains 3 files to add to your jaecyk/NTB-predictor repo root:

Procfile — tells Railway how to start your server
railway.json — Railway deployment config
backend_db.py — updated to support Railway PostgreSQL (replaces existing file)

Also rename requirements_backend.txt to requirements.txt in your repo.
Step 2 — Deploy on Railway
1. Go to railway.app → New Project
2. Choose Deploy from GitHub repo
3. Select jaecyk/NTB-predictor
4. Railway detects Python automatically — click Deploy
5. In your project: click + Add → Database → PostgreSQL
6. Railway auto-sets DATABASE_URL in your service env
7. Add env var: ALLOWED_ORIGINS=*
8. Wait ~2 minutes → your API is live
Step 3 — Connect API to this dashboard
Copy your Railway URL (looks like https://ntb-predictor-production-xxxx.up.railway.app) and paste it in the NTB ML Predict tab.
API Endpoints your model exposes
GET /health — check models loaded (91d, 182d, 364d)
POST /snapshots — save a market snapshot (14 features)
GET /snapshots/latest — get latest snapshot per tenor
POST /predict/latest — run PKL model → returns stop rate
GET /predictions/history — past predictions log

This dashboard calls /snapshots then /predict/latest when you click Run.
Alternative — Render.com (if Railway doesn't work)
1. render.com → New Web Service → Connect GitHub
2. Build command: pip install -r requirements_backend.txt
3. Start command: uvicorn backend_main_hosted:app --host 0.0.0.0 --port $PORT
4. Add PostgreSQL database plugin
5. Set ALLOWED_ORIGINS=* env var
Note: Render free tier sleeps after 15min inactivity.
